🧠

2brain

A file-based, multi-domain, human-approved AI knowledge wiki.

Raw sources are digested by an agent into candidate Markdown pages. You review each page and approve it into a persistent, Git-backed knowledge tree. Every page carries visible confidence, evidence, and inline contradictions — knowledge compounds instead of being re-derived from scratch on every query.

The Core Idea

Classic RAG re-reads raw documents on every question. 2brain keeps a synthesized wiki that an agent grows over time. Cross-references, contradictions, and summaries are already in the page by the time you ask. The human stays in the loop at one — and only one — choke point: approving candidate pages.

What Each Tab Does

The nav is organized into four color-coded groups — Read · Pipeline · Ops · Meta. Each group has its own accent hue that matches the tab underline in the header.

📘 Read — consume knowledge

📖 Wiki

Browse approved pages grouped by type (topics, concepts, entities, sources, reports, questions). List / cards / compact views with title, date, and tag filters. Click a page to read it with its metadata sidebar, sources, tags, and review trail.

💬 Ask

Grounded LLM chat. Uses ask_llm() to answer a question from the approved wiki (optionally including candidates), structured into the four-section CLAUDE.md format with [APPROVED]/[CANDIDATE] citations. Ephemeral — nothing gets filed.

🔍 Query

Keyword search across approved pages and pending candidates. Type/tag/status chips, min-confidence slider, scope toggle. Each result is ranked by title/body/tag hits and labeled with its status badge.

🛠 Pipeline — build knowledge

📥 Ingest

Add a raw source by pasting text or fetching a URL (GitHub repo URLs resolve to README + top-level tree). 2brain stores an immutable copy under inbox/raw/<raw_id>/ with full metadata. The same tab lists every raw source ever ingested — click in to read it, jump to digesting, or delete.

⚙️ Digest

Pick a raw source → the Claude agent reads the domain's schema.md, checks for near-duplicates, and produces candidate pages with required frontmatter and per-claim evidence. For a backlog, use the digestAgent on the Agents tab — same pipeline, but batch-drains inbox/raw/ by domain_hint with a seen ledger.

✅ Review

The queue of pending candidates. Each candidate shows its rendered body, target path, confidence, and proposed operation (create / update / replace / merge / archive / move / split). Approve, reject, or edit the raw Markdown. Approval can cascade-drop the cited raw sources.

🧰 Ops — operate the system

🗂️ Jobs

Every digest / research / lint / agent run, recorded in jobs/ with full event log. Grouped by state (running / completed / failed) with bulk-delete controls. Auto-refreshes while a job is running.

🩺 Health

Runs the lint scanner: low-confidence pages, unresolved contradictions, orphans, stale pages, stale candidates, stuck jobs, and index mismatches. Regenerates the auto-indexes on every run.

⚙️ Config

Edit config/app.yaml from the browser — domain, source types, suggested tags, digest limits, lint thresholds, theme. Unknown keys round-trip so manual edits survive.

🧭 Meta — agents & info

🤖 Agents

Registered background agents — each lives in agents/<name>/ with its own config.yaml, prompt.md, and seen.json. Currently shipping: digestAgent (batch digester), deepSearch (files research reports as candidates), lintAgent, conflicAgent, sourceDiscovery. Each can be scheduled (off / manual / hourly / daily / weekly), run on demand, and has its seen ledger reset from the UI.

ℹ️ About

This page — the system explainer, quick start, rules the agent follows, and a reference for candidate operations and confidence scoring.

How Confidence Works

Every page, every claim, and every search result carries a numeric score from 0.00 to 1.00. The agent assigns it using a rubric — direct quotes from primary sources score high, inferences score lower, contradicted claims drop further.

Very high0.90 – 1.00 · Strong source support
High0.75 – 0.89 · Good support, minor uncertainty
Medium0.55 – 0.74 · Useful but needs review/context
Low0.35 – 0.54 · Weak, incomplete, or inferred
Very low0.00 – 0.34 · Speculative or unresolved

Candidate Operations

A candidate doesn't always propose a new page. It can propose any change to the tree:

create

Add a new page at target_path.

update

Replace an existing page with a revised version.

replace

Like update, but signals a significant rewrite.

merge

Reviewer manually merges this candidate into an existing page.

archive

Retire an approved page to .archive/.

move

Relocate an approved page to a new path or domain.

split

Split one page into several (approve each split separately).

What Makes 2brain Different

  • Human approval gate. Nothing lands in the trusted tree without you clicking Approve.
  • Always-visible confidence. Every page, citation, and search result carries a score.
  • Inline contradictions. Conflicts live inside the relevant page as [!contradiction] callouts — not in a separate tracker you'll forget to read.
  • Evidence-required claims. Every Key Claim must cite a source. No evidence-free statements.
  • Multi-domain. Each domain has its own schema and evolves independently.
  • Git-backed. Raw sources, candidates, and approved pages are all versioned. No separate history store.
  • Deep research. Research agents investigate across approved pages, candidates, raw sources — and file their output as a candidate for approval.
  • File-backed agents. Every agent is a folder under agents/<name>/ with its own config.yaml, prompt.md, and seen.json — user-editable from the UI or on disk.

A Quick Start

  1. Ingest a source — paste an article, fetch a URL, or drop in a GitHub repo. The same tab lists every raw you've ingested.
  2. Digest it — pick a target domain and let Claude draft a candidate page. For backlogs, schedule the digestAgent instead.
  3. Review the queue — read each candidate and approve, edit, or reject.
  4. Browse the wiki, ask a question, or search — approved knowledge compounds over time.
  5. Run a health check periodically to find drift, orphans, and stale content.

Rules the Agent Always Follows

  • Never modifies raw sources.
  • Never promotes candidates to approved without human action.
  • Always shows confidence.
  • Always labels candidate content as [CANDIDATE].
  • Never mixes pages across domains without an explicit move.
  • Reads the domain schema.md before writing any page.
  • Checks for near-duplicates before proposing a new page.
